# 个人 AI 生图控制台部署文档

## 部署目标

这是一个 Vite 构建的纯前端静态应用。生产环境不需要 Node 常驻服务，构建后部署 `dist/` 即可。

GitHub 仓库：<https://github.com/cnxmu/GPT-Image-Web>

由于 API Key 保存在浏览器 IndexedDB，推荐把它部署成你自己的个人工具，而不是完全公开给不可信访客使用。

## 先构建

```bash
npm install
npm run build
```

构建结果输出到：

```text
dist/
```

本地预览：

```bash
npm run preview
```

## 上传到生产服务器时，到底要传什么

只需要上传 `dist/` 目录里的内容。

如果你的网站根目录是：

```text
/www/wwwroot/img-web/
```

那就把 `dist/` 里面的文件和文件夹上传进去，最终这个目录里应该直接能看到：

- `index.html`
- `assets/`
- `ai-img.svg`
- `manifest.webmanifest`
- `robots.txt`
- `sitemap.xml`
- `og-image.png`

不需要上传这些开发文件：

- `src/`
- `public/`
- `node_modules/`
- `.git/`
- 测试文件
- 本地文档

`deploy/` 目录中的文件是参考配置，不是运行时必须上传的前端资源。

## 站点路径要求

项目当前按根路径站点部署设计，例如：

```text
https://your-domain.example/
```

没有配置 Vite 子路径 `base`。如果你要部署到子目录，例如 `https://example.com/app/`，需要额外改 Vite 基础路径和相关静态资源引用。

## Nginx 部署示例

下面是一份适合静态站点的基础示例：

```nginx
server {
  listen 80;
  server_name your-domain.example;
  return 301 https://$host$request_uri;
}

server {
  listen 443 ssl http2;
  server_name your-domain.example;

  root /www/wwwroot/img-web;
  index index.html;

  ssl_certificate /path/to/fullchain.pem;
  ssl_certificate_key /path/to/privkey.pem;
  ssl_protocols TLSv1.2 TLSv1.3;

  add_header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data: blob: https:; connect-src 'self' https://img.xmu.la; font-src 'self' data:; object-src 'none'; base-uri 'self'; frame-ancestors 'none'; form-action 'self'; upgrade-insecure-requests" always;
  add_header Referrer-Policy "strict-origin-when-cross-origin" always;
  add_header Permissions-Policy "camera=(), microphone=(), geolocation=(), payment=(), usb=(), serial=(), clipboard-read=(), clipboard-write=(self)" always;
  add_header X-Content-Type-Options "nosniff" always;
  add_header X-Frame-Options "DENY" always;
  add_header Cross-Origin-Opener-Policy "same-origin" always;

  location / {
    try_files $uri $uri/ /index.html;
  }

  location ~* \.(js|css)$ {
    expires 12h;
    access_log off;
  }

  location ~* \.(png|jpg|jpeg|webp|svg|ico)$ {
    expires 30d;
    access_log off;
  }
}
```

如果你不想把安全头手写进主配置，也可以参考仓库里的片段文件：

```text
deploy/nginx-security-headers.conf
```

## HTTPS 和访问控制

个人部署时建议至少启用：

- HTTPS
- 访问控制，例如 Basic Auth、VPN、Cloudflare Access、Tailscale 或仅内网访问
- 服务器防火墙，仅开放必要端口

Nginx Basic Auth 示例：

```nginx
auth_basic "Private Image Console";
auth_basic_user_file /etc/nginx/.htpasswd;
```

生成 `.htpasswd` 示例：

```bash
sudo apt install apache2-utils
sudo htpasswd -c /etc/nginx/.htpasswd your_name
```

## API Key 与接口说明

当前前端会直接请求：

```text
https://img.xmu.la
```

其中：

- `gpt-image-2` 走 `/v1/images/generations` 和 `/v1/images/edits`
- `nano-banana-*` 走 `/v1/chat/completions`
- Agent 对话走 `/v1/responses`

当前密钥策略：

- 生图 API Key 由你在网页里手动填写
- Nano Banana 可使用单独 API Key；留空时回退使用普通生图 Key
- Agent API Key 由你在网页里手动填写
- 所有这些 Key 都只保存在当前浏览器 IndexedDB

不要把 Key 写进：

- 源码
- `index.html`
- 静态 JS
- Nginx 配置
- 公开环境变量

## SEO 配置

项目内已包含基础 SEO：

- `index.html`
- `public/robots.txt`
- `public/sitemap.xml`
- `public/manifest.webmanifest`
- `public/og-image.png`

当前 `robots.txt` 和 `sitemap.xml` 仍使用占位域名：

```text
https://example.com/
```

上线前请先修改这些文件：

- `public/robots.txt`
- `public/sitemap.xml`

修改后重新执行一次：

```bash
npm run build
```

如果你希望社交平台稳定抓取分享图，也可以把 `index.html` 里的 `og:image` 和 `twitter:image` 改为正式域名绝对地址。

如果这是一个私有个人工具，不希望被搜索引擎收录，也可以把 `robots.txt` 改成：

```text
User-agent: *
Disallow: /
```

## 本地数据存储

浏览器 IndexedDB 会保存：

- 生图 API Key
- Nano Banana API Key
- Agent API Key
- 设置项
- 我的历史
- 个人模板
- 图生图参考图
- Agent 对话图片
- 新生成结果图片

设置中可执行：

- 清空我的历史
- 清理未引用图片
- 清空个人模板
- 清除我的本地数据

“清理未引用图片”只删除没有被历史或 Agent 对话引用的本地孤立图片，不会破坏仍能回显的历史结果。

## 并发与费用建议

默认生成并发上限为 `20`，可在设置中调整到 `1-100`。

建议：

- 日常使用保持默认 `20`
- 如果服务端不稳定，适当调低并发
- 批量生成超过 `20` 张时，应用会先弹确认
- Nano Banana 如果偶发 `524`，优先把模型切到 `1K`、数量改成 `1`、减少参考图

## 上线后检查清单

- `npm run test` 已通过
- `npm run lint` 已通过
- `npm run build` 已通过
- 站点目录已上传 `dist/` 内容
- HTTPS 已开启
- `try_files $uri $uri/ /index.html` 已配置
- 安全响应头已配置
- `robots.txt` 和 `sitemap.xml` 已替换为正式域名
- 如需分享卡片，`og:image` / `twitter:image` 已改成正式域名绝对地址
- 站点里没有硬编码 API Key
- 首次打开网站后，已在“个人设置”中保存自己的密钥
