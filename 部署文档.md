# 个人 AI 生图控制台部署文档

## 部署方式

这是一个 Vite 构建的纯前端静态应用。生产环境不需要 Node 常驻服务，构建后上传 `dist/` 目录即可。

GitHub 仓库：<https://github.com/cnxmu/GPT-Image-Web>

## 构建

在项目根目录执行：

```bash
npm install
npm run build
```

构建产物输出到：

```text
dist/
```

本地预览生产包：

```bash
npm run preview
```

## 需要上传哪些文件

只上传 `dist/` 目录里的内容。

如果服务器站点根目录是：

```text
/www/wwwroot/img-web/
```

那么最终目录里应该直接能看到：

- `index.html`
- `assets/`
- `ai-img.svg`
- `manifest.webmanifest`
- `robots.txt`
- `sitemap.xml`
- `og-image.png`

不需要上传：

- `src/`
- `public/`
- `node_modules/`
- `.git/`
- 测试文件
- 本地 Markdown 文档
- `package.json`
- `package-lock.json`

`deploy/` 目录是参考配置，不是前端运行必须文件。

## 站点路径

项目当前按根路径部署设计，例如：

```text
https://tu.xmu.la/
```

如果要部署到子目录，例如 `https://example.com/app/`，需要额外调整 Vite 的 `base` 和静态资源路径。

## Nginx 示例

下面是完整静态站点示例，请把域名、证书路径和 root 改成自己的：

```nginx
server {
  listen 80;
  server_name your-domain.example;
  return 301 https://$host$request_uri;
}

server {
  listen 443 ssl http2;
  server_name your-domain.example;

  root /www/wwwroot/img-web;
  index index.html;

  ssl_certificate /path/to/fullchain.pem;
  ssl_certificate_key /path/to/privkey.pem;
  ssl_protocols TLSv1.2 TLSv1.3;

  add_header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data: blob: https:; connect-src 'self' https://api.google.com; font-src 'self' data:; object-src 'none'; base-uri 'self'; frame-ancestors 'none'; form-action 'self'; upgrade-insecure-requests" always;
  add_header Referrer-Policy "strict-origin-when-cross-origin" always;
  add_header Permissions-Policy "camera=(), microphone=(), geolocation=(), payment=(), usb=(), serial=(), clipboard-read=(), clipboard-write=(self)" always;
  add_header X-Content-Type-Options "nosniff" always;
  add_header X-Frame-Options "DENY" always;
  add_header Cross-Origin-Opener-Policy "same-origin" always;

  location / {
    try_files $uri $uri/ /index.html;
  }

  location ~* \.(js|css)$ {
    expires 12h;
    access_log off;
  }

  location ~* \.(png|jpg|jpeg|webp|svg|ico|webmanifest)$ {
    expires 30d;
    access_log off;
  }
}
```

仓库中也提供了安全头参考片段：

```text
deploy/nginx-security-headers.conf
```

## API 端点

当前接口基地址在：

[src/lib/constants.ts](E:/Users/cnxmu/Documents/kaifa/生图工作站-网页端/src/lib/constants.ts:1)

```ts
export const API_BASE_URL = 'https://api.google.com'
```

当前会请求：

- `POST https://api.google.com/v1/images/generations`
- `POST https://api.google.com/v1/images/edits`
- `POST https://api.google.com/v1beta/models/nano-banana-2:generateContent`
- `POST https://api.google.com/v1beta/models/nano-banana-pro:generateContent`
- `POST https://api.google.com/v1/responses`

模型对应关系：

- `gpt-image-2` 文生图走 `/v1/images/generations`。
- `gpt-image-2` 图生图走 `/v1/images/edits`。
- `nano-banana-2` / `nano-banana-pro` 文生图和图生图都走 `/v1beta/models/{model}:generateContent`。
- Nano Banana 的构图比例会传到 `generationConfig.imageConfig.aspectRatio`，分辨率档位会传到 `generationConfig.imageConfig.imageSize`。
- Nano Banana 的 `responseModalities` 固定为 `["IMAGE"]`，`thinkingConfig.thinkingLevel` 固定为 `minimal`，`tools` 固定为空数组。
- Nano Banana 不发送 `quality`、`moderation`、`output_format`、`output_compression`。
- Nano Banana 图生图会把参考图原始文件作为 `inlineData` 发送，不压缩、不转码。

如果你改了 `API_BASE_URL`，Nginx 的 CSP 也要同步修改 `connect-src`，否则浏览器会拦截请求。

修改后重新构建并重新上传 `dist/`。

## API Key

项目不会把 API Key 写进源码或构建产物。

用户在网页里填写：

- 生图 API Key
- Nano Banana API Key
- Agent API Key

这些 Key 只保存在当前浏览器 IndexedDB。清除浏览器站点数据后会一起删除。

Key 用途：

- 生图 API Key：用于 `gpt-image-2`。
- Nano Banana API Key：用于 `nano-banana-2` / `nano-banana-pro`。
- Agent API Key：用于个人 Agent 对话和提示词优化。

## 本地数据存储

浏览器 IndexedDB 数据库名：

```text
ImageWorkbenchDB
```

会保存：

- API Key
- 设置项
- 个人模板
- 我的历史
- 图生图参考图
- Agent 会话
- Agent 上传图片
- 生成结果图片

生成结果保存成功后，结果卡片、预览弹窗和历史回显会优先使用本地 Blob 资产生成的 `blob:` 地址。接口返回的远程图片 URL 主要用于首次读取尺寸和写入本地资产。


## 访问控制建议

这是个人工具，建议生产环境至少做到：

- 使用 HTTPS
- 不公开给不可信用户
- 可选加 Basic Auth、VPN、Cloudflare Access、Tailscale 或仅内网访问
- 不把 API Key 写进 Nginx 配置或静态文件

Basic Auth 示例：

```nginx
auth_basic "Private Image Console";
auth_basic_user_file /etc/nginx/.htpasswd;
```

生成 `.htpasswd` 示例：

```bash
sudo apt install apache2-utils
sudo htpasswd -c /etc/nginx/.htpasswd your_name
```

## 并发与资源

默认并发是 `10`，可在网页个人设置中调整为 `1-10`。超过并发上限的图片会自动排队等待。

建议：

- 日常保持默认值。
- 如果上游容易限流或失败，降低并发。
- 一次生成超过 `20` 张时，页面会弹出确认。
- 长期使用后，可以在个人设置中清理旧历史、旧 Agent 会话和未引用图片。
