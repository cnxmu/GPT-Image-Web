# 生图工作站部署文档

## 部署目标

本项目是 Vite 构建的纯前端静态应用。生产构建后只需要部署 `dist/` 目录即可。

由于 API Key 保存在浏览器 IndexedDB，建议部署为个人私有工具，不建议直接开放给不可信公众用户。

## 构建

```bash
npm install
npm run build
```

构建产物位于：

```text
dist/
```

本地预览：

```bash
npm run preview
```

## Nginx 部署示例

示例站点配置：

```nginx
server {
  listen 443 ssl http2;
  server_name example.com;

  root /var/www/image-workbench/dist;
  index index.html;

  ssl_certificate /path/to/fullchain.pem;
  ssl_certificate_key /path/to/privkey.pem;

  include /var/www/image-workbench/deploy/nginx-security-headers.conf;
}
```

项目已提供安全头片段：

```text
deploy/nginx-security-headers.conf
```

该配置包含：

- Content-Security-Policy
- Referrer-Policy
- Permissions-Policy
- X-Content-Type-Options
- X-Frame-Options
- Cross-Origin-Opener-Policy
- SPA fallback：`try_files $uri $uri/ /index.html`

## HTTPS 和访问控制

个人部署时至少启用：

- HTTPS。
- 访问控制，例如 Basic Auth、VPN、Cloudflare Access、Tailscale 或仅内网访问。
- 服务器防火墙，仅开放必要端口。

Nginx Basic Auth 示例：

```nginx
auth_basic "Image Workbench";
auth_basic_user_file /etc/nginx/.htpasswd;
```

生成 `.htpasswd` 可使用：

```bash
sudo apt install apache2-utils
sudo htpasswd -c /etc/nginx/.htpasswd your_name
```

## API Key 策略

当前版本：

- 生图 API Key 和 Agent API Key 由用户在浏览器内输入。
- 密钥保存到当前浏览器 IndexedDB。
- 前端请求 `https://img.xmu.la` 时使用 `Authorization: Bearer <key>`。

注意：

- 不要把 API Key 写进源码。
- 不要把 API Key 写进 `index.html`、静态 JS、Nginx 配置或公开环境变量。
- 共享电脑或不可信浏览器环境不适合保存密钥。

更高安全方案：

- 自建后端代理或边缘函数。
- 真实 API Key 只保存在服务端。
- 前端只调用自己的受保护接口。
- 服务端增加用户认证、限流、审计和额度控制。

## 数据存储与清理

浏览器 IndexedDB 会保存：

- API Key。
- 设置项。
- 模板。
- 历史记录。
- 图生图参考图资产。
- Agent 对话图片资产。

应用设置中提供：

- 清除密钥。
- 清空历史与历史图片。
- 清空资源缓存。
- 清空模板。
- 清除全部本地数据。

如果浏览器清除站点数据，上述本地数据会被删除。

## 并发与费用保护

默认生成并发上限为 `20`，可在设置中调整到 `1-100`。

建议：

- 日常使用保持默认 `20`。
- 网络或服务端限流明显时调低。
- 批量生成超过 `20` 张时，应用会弹出确认提示。
- 并发越高，失败率、浏览器内存压力和费用风险越高。

## 部署检查清单

- `npm run test` 通过。
- `npm run build` 通过。
- `npm run lint` 通过。
- `npm audit` 无高危漏洞。
- `dist/` 已部署。
- HTTPS 已开启。
- 访问控制已开启。
- 安全响应头已配置。
- 确认站点没有内置 API Key。
- 在浏览器设置页保存个人 API Key 后再使用。
