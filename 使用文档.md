# 个人 AI 生图控制台使用文档

## 第一次打开

1. 打开网页。
2. 点击右上角“个人设置”。
3. 填写“生图 API Key”。
4. 填写“Agent API Key”。
5. 选择 Agent 模型。
6. 按需调整“生成并发上限”，默认是 `20`。
7. 保存后关闭设置。

密钥只保存在当前浏览器 IndexedDB。换浏览器、换设备或清除站点数据后，需要重新填写。

## 文生图

1. 在右侧“个人生成设置”里选择“文生图”。
2. 在中间“编辑提示词”中填写提示词。
3. 可选填写负面提示词。
4. 选择构图比例、分辨率档位、质量、审查、输出格式、数量和压缩率。
5. 点击“开始我的生成”。

说明：

- 生图模型固定为 `gpt-image-2`。
- 单次接口请求固定 `n=1`。
- 数量大于 1 时，会由本地队列逐张提交。
- 数量超过 `20` 时会先弹出确认，避免误触发大量请求。
- PNG 不会传压缩率；JPEG / WebP 会传 `output_compression`。

## 图生图

1. 在右侧“个人生成设置”里选择“图生图”。
2. 上传至少一张参考图。
3. 填写提示词和负面提示词。
4. 设置生成参数。
5. 点击“开始我的生成”。

参考图规则：

- 支持 PNG、JPEG、WEBP。
- 不支持 GIF 和 SVG。
- 原图保存、原图上传，不压缩、不转码。
- 可以拖拽上传，也可以点击上传。
- 参考图缩略图可以预览。
- 上传超过 4 张时，右侧只显示前 4 张，其余可在“管理全部”里查看。
- 可一键“清空”当前已上传的全部参考图。

## 结果区

“我的结果”只显示当前可见批次。

结果卡片会显示：

- 排队中
- 生成中和实时耗时
- 成功结果
- 失败错误
- 实际尺寸
- 预览、下载、复制

每次开始新一轮生成时，结果区会切到新批次。旧批次不会删除，会保存在“我的历史”里。

生成成功后，图片会保存为当前浏览器里的本地 Blob 资产。保存成功后，结果卡片、预览弹窗和历史回显会优先使用本地 `blob:` 地址，减少重复请求远程图片 URL。

## 我的历史

左侧“我的资源”里切到“我的历史”即可查看生成记录。

历史支持：

- 全部 / 成功 / 失败筛选
- 点击历史回填提示词和参数
- 点击历史在结果区回显这次结果
- 删除单条历史
- 清空所有历史

删除历史时，会同步删除这条历史引用的本地图片资产。

如果刷新页面时还有任务没完成，应用会在下次打开时尝试恢复 `running` 历史，把未完成图片重新入队继续生成。

## 个人模板

左侧“我的资源”里切到“个人模板”即可管理模板。

模板分两类：

- 个人模板：你自己保存的模板，可删除。
- 内置预设：系统内置模板，默认折叠，不可删除。

当前内置预设包括：

- 个人头像预设
- 个人壁纸预设
- 个人电商图预设
- 个人产品海报预设
- 公众号封面个人预设
- 视频封面个人预设
- 小红书封面个人预设
- 内容头图个人预设
- 短视频竖版个人预设

保存个人模板：

1. 调好当前提示词和右侧参数。
2. 在模板区域填写名称和描述。
3. 点击“保存为个人模板”。

点击模板卡片会把模板参数回填到当前表单。

## 提示词优化

在“我的提示词”区域选择“编辑提示词”，点击“帮我优化”。

优化结果会包含：

- 优化后的提示词
- 优化后的负面提示词
- 和原提示词的改进说明

点击“使用这版”后才会写回当前表单。

## 个人 Agent

在“我的提示词”区域切到“个人 Agent”。

Agent 可以帮助你：

- 分析图片
- 改写提示词
- 建议参数改动
- 应用模板
- 创建个人模板
- 复用历史结果
- 解释失败原因
- 确认后开始生成

Agent 的所有动作都会先显示卡片。会改表单、创建模板、展示历史或触发生图的动作，都必须你点击确认才会执行。

## Agent 图片上传

Agent 对话支持文字 + 图片，也支持只发图片。

规则：

- 支持 PNG、JPEG、WEBP。
- 不支持 GIF 和 SVG。
- 单条消息最多 8 张图片。
- 图片本地保存不压缩、不转码。
- 发送给 Agent 时会转成 base64 data URL。
- 单张图片超过 10MB 时会弹出确认。

图片操作：

- 待发送图片可以移除。
- 待发送图片和已发送图片都可以预览。
- 已发送图片可以“加入参考图”。
- 加入参考图后，会自动切换到图生图模式。

## Agent 多会话

Agent 对话保存在当前浏览器 IndexedDB。

可用操作：

- “新对话”：创建一个空会话。
- “我的会话”：打开历史会话弹窗。
- 重命名会话。
- 删除会话。
- 清空当前对话。

删除会话或清空会话时，会同步删除该会话引用的本地图片资产。

## 个人设置

个人设置里可以管理：

- 生图 API Key
- Agent API Key
- Agent 模型
- 生成并发上限
- 本地数据整理

本地数据整理支持：

- 保留最近 20 / 50 / 100 条生成历史
- 保留最近 20 / 50 / 100 条 Agent 会话
- 清理未引用图片
- 清空我的历史
- 清空个人模板
- 清除我的本地数据

“清理未引用图片”只会删除没有被历史或 Agent 会话引用的孤立图片资产，不会破坏仍可回显的历史结果或对话图片。

## 常见问题

### 为什么结果区看不到上一轮图片？

这是当前设计。结果区只显示当前批次，上一轮图片在“我的历史”里，点击历史即可回显。

### 为什么 Network 里生成后还会看到图片请求？

生成完成后，前端需要读取真实尺寸并把图片保存成本地 Blob。保存成功后，结果展示和预览会优先使用本地 `blob:` 地址。

### 为什么刷新后运行中任务会继续？

刷新会中断浏览器中的原请求。应用会读取本地 `running` 历史，把未完成 job 重新入队继续生成，因此可能产生新的接口请求。

### 为什么 Agent 上传大图会确认？

Agent 请求会把图片转成 base64 data URL。图片太大时请求体会明显变大，所以超过 10MB 会先确认。

### 密钥会写进代码吗？

不会。密钥只保存在当前浏览器 IndexedDB，不会写进源码或构建产物。
