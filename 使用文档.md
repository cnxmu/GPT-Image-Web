# 个人 AI 生图控制台使用文档

## 首次使用

1. 打开网页。
2. 点击右上角“个人设置”。
3. 填写“生图 API Key”。
4. 按需填写 “Nano Banana API Key（可选）”。
5. 填写 “Agent API Key”。
6. 选择 Agent 模型，可选 `gpt-5.5`、`gpt-5.4`、`gpt-5.4-mini`。
7. 按需调整“生成并发上限”，默认 `20`。
8. 保存后关闭设置。

密钥、历史、模板、Agent 会话和图片资产仅保存在当前浏览器本地 IndexedDB。

## 如何修改接口地址

当前版本不能在网页设置里直接修改 API 端点。

如果你要更换接口地址，需要修改代码文件：

[src/lib/constants.ts](E:/Users/cnxmu/Documents/kaifa/生图工作站-网页端/src/lib/constants.ts:1)

把：

```ts
export const API_BASE_URL = 'https://www.www.com'
```

改成你自己的接口域名，例如：

```ts
export const API_BASE_URL = 'https://你的接口域名'
```

说明：

- 这里只改一处就够了
- 文生图、图生图、Nano Banana、Agent 对话都会一起使用新的接口地址
- 改完后需要重新构建项目
- 如果站点已经上线，还要把新的 `dist/` 重新上传到服务器

重新构建命令：

```bash
npm run build
```

## 开始前先了解模型差异

### `gpt-image-2`

右侧参数区会显示：

- 构图比例
- 分辨率档位
- 最终尺寸
- 质量
- 审查
- 输出格式
- 数量
- 压缩率

### `nano-banana-2` / `nano-banana-pro`

右侧参数区会显示：

- 构图比例
- 当前模型
- 数量
- 最大 Token
- 采样温度
- 核采样
- 采样种子

“Nano Banana 详细模型”默认关闭。关闭时只会使用：

- `nano-banana-2`
- `nano-banana-pro`

如果你的上游重新支持详细模型，可以在“个人设置”里开启 “Nano Banana 详细模型”。开启后右侧参数区会显示“详细模型”，可选择 `1K`、`2K`、`4K` 等模型。

不会显示这些 `gpt-image-2` 专属参数：

- 分辨率档位
- 最终尺寸
- 质量
- 审查
- 输出格式
- 压缩率

## 文生图

1. 在右侧“参数”中选择“文生图”。
2. 选择生图模型。
3. 在中间“编辑提示词”中填写“提示词”和“负面提示词”。
4. 按当前模型显示的参数面板完成设置。
5. 点击“开始我的生成”。

说明：

- 批量生成会拆成多次请求，每次请求固定 `n=1`。
- 数量超过 `20` 时会弹出确认，避免误触发大量请求。
- Nano Banana 模型会走单独的聊天式图片接口，不使用 `gpt-image-2` 的输出格式和质量参数。

## 图生图

1. 在右侧“参数”中选择“图生图”。
2. 上传至少一张参考图。
3. 填写提示词和负面提示词。
4. 按当前模型完成参数设置。
5. 点击“开始我的生成”。

参考图规则：

- 仅支持 PNG、JPEG、WEBP。
- 不支持 GIF 和 SVG。
- 原图保存、原图上传，不压缩、不转码。
- 点击缩略图或预览按钮可以打开大图预览。

## 结果区

结果区只显示当前可见批次，包含：

- 排队中
- 生成中和实时耗时
- 成功结果
- 失败错误
- 实际尺寸
- 下载、复制、预览

每次新生成时，结果区只切到新批次。旧批次不会丢失，而是进入“我的历史”。

## 我的历史

左侧“我的历史”中可以查看你自己的生成记录。

点击某条历史会：

- 回填提示词和参数
- 在结果区显示这条历史的结果
- 如果该记录原本还在运行，会优先显示真实运行中的 live 批次

删除历史时，会同步删除该历史引用的本地图片资产。

新生成成功的图片会以本地 Blob 资产形式保存在 IndexedDB；历史记录本身只保存引用，因此更适合长期使用。

## 模板

左侧“个人模板”区域分成两组：

- 个人模板：你自己保存的模板，默认展开，可删除
- 系统预设：默认折叠，不可删除

当前内置预设包括：

- 个人头像预设
- 个人壁纸预设
- 个人电商图预设
- 个人产品海报预设
- 公众号封面个人预设
- 视频封面个人预设
- 小红书封面个人预设
- 内容头图个人预设
- 短视频竖版个人预设

保存为个人模板：

1. 调好提示词和参数。
2. 在左侧模板区填写名称和描述。
3. 点击“保存为个人模板”。

## 提示词优化

在“编辑提示词”标签下点击“帮我优化”，可以让 Agent 优化当前提示词。

返回内容包括：

- 优化后提示词
- 优化后负面提示词
- 简短对比说明

点击“使用这版”后，会直接写回当前表单。

## 个人 Agent 对话

切换到“个人 Agent”标签后，可以像聊天一样让 Agent 协助你的生图流程。

Agent 可以提出需要你确认的动作：

- 修改表单参数
- 应用模板
- 保存为个人模板
- 展示历史结果
- 应用建议并开始生成
- 解释失败原因、参数差异或结果差异

所有会改表单、创建模板或触发生图的动作，都必须你亲自点击确认才会执行。

## Agent 图片上传与分析

Agent 对话支持上传图片分析。

规则：

- 支持 PNG、JPEG、WEBP
- 不支持 GIF 和 SVG
- 单条消息最多 8 张图片
- 本地保存不压缩、不转码
- 发送给 Agent 时会转为 base64 data URL
- 单张图片超过 10MB 时会弹出确认提示

图片操作：

- 点击缩略图可打开大图预览
- 已发送图片可点击“加入参考图”
- 加入参考图后会自动切换到图生图模式

## Agent 多会话

个人 Agent 支持多个本地会话：

- `新对话`：创建空白会话
- `我的会话`：打开历史会话弹窗
- `重命名我的对话`：修改标题
- `删除我的对话`：删除该会话及其引用图片
- `清空当前对话`：只清空当前会话消息和图片

会话会保存在 IndexedDB 中，刷新页面后仍然存在。

## 设置和清理

设置中可以：

- 保存或清除密钥
- 切换 Agent 模型
- 调整生成并发上限
- 开关 Nano Banana 详细模型
- 清空我的历史
- 清理未引用图片
- 清空个人模板
- 清除我的本地数据

“清理未引用图片”只会删除没有被历史或 Agent 对话引用的孤立图片资产，不会破坏仍可回显的历史结果、参考图或对话图片。

## 常见问题

### 为什么新生成时旧结果不显示？

这是当前设计。结果区只显示当前批次，旧结果统一放在“我的历史”里，点击历史即可重新显示。

### 为什么批量生成会提示确认？

超过 `20` 张图片时，请求数和费用都会明显增加，也更容易失败，所以会先确认一次。

### 为什么刷新后运行中任务会继续？

刷新会中断浏览器里的原始请求。应用会读取本地 `running` 历史，把未完成任务重新入队续跑，因此可能产生新的接口请求。

### 为什么 Nano Banana 会报 `524`？

这通常表示中转站等待上游模型超时。可以先试：

- 使用 `1K` 模型
- 数量改成 `1`
- 减少参考图
- 稍后重试

### 为什么 Agent 上传大图会提示确认？

Agent 请求会把图片转成 base64 data URL。图片太大时，请求体会明显变大，可能更慢或更容易失败。

### 密钥保存在哪里？

保存在当前浏览器 IndexedDB，不会写进项目代码或构建产物。清除浏览器站点数据后会一起删除。

### 清理未引用图片会删除历史图片吗？

不会。它只删除没有被历史或 Agent 对话引用的孤立图片资产。要删除历史图片，请删除对应历史，或使用“清空我的历史”。
