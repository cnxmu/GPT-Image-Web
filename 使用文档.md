# 个人 AI 生图控制台使用文档

## 首次使用

1. 打开网页。
2. 点击右上角“设置”。
3. 填写“生图 API Key”和“Agent API Key”。
4. 选择 Agent 模型，可选 `gpt-5.5`、`gpt-5.4`、`gpt-5.4-mini`。
5. 按需调整“生成并发上限”，默认 `20`。
6. 保存后关闭设置。

密钥仅保存在当前浏览器本地 IndexedDB。

## 文生图

1. 在右侧“个人生成设置”中选择“文生图”。
2. 在中间“编辑提示词”里填写“提示词”和“负面提示词”。
3. 设置构图比例、分辨率档位、质量、审查、输出格式、数量和压缩率。
4. 点击“开始我的生成”。

说明：

- 生图模型可在右侧参数中选择，支持 `gpt-image-2`、`nano-banana-2`、`nano-banana-pro`。
- 批量生成会拆成多次请求，每次请求固定 `n=1`。
- 数量超过 `20` 时会弹出确认，避免误触发大量费用。

## 图生图

1. 在右侧“个人生成设置”中选择“图生图”。
2. 上传至少一张参考图。
3. 填写提示词和负面提示词。
4. 设置生成参数。
5. 点击“开始我的生成”。

参考图规则：

- 仅支持 PNG、JPEG、WEBP。
- 不支持 GIF 和 SVG。
- 原图保存、原图上传，不压缩、不转码。
- 点击参考图缩略图或预览按钮可以打开大图预览。

## 结果区

结果区显示当前可见批次：

- 排队中。
- 生成中和实时耗时。
- 成功结果。
- 失败错误。
- 实际尺寸。
- 下载、复制、预览。

每次新生成会只显示新批次。旧批次仍会写入我的历史，点击历史可以再次显示。

## 我的历史

左侧“我的历史”中可以查看你自己的生成记录。

点击我的历史会：

- 回填左侧/中间表单参数。
- 在结果区显示该历史的图片、耗时、尺寸和错误。
- 如果这次生成仍在运行，会显示真实运行中的 live 批次。

删除历史会同步删除该历史引用的本地图片资产。

新生成成功的图片会以 Blob 资产形式保存在 IndexedDB，我的历史只保存本地资产引用。这样可以减少历史本身的体积，也能继续支持历史结果回显、下载、复制和预览。

## 模板

左侧“模板”分为：

- 个人模板：你保存或 Agent 创建的模板，默认展开，可删除。
- 内置预设：默认折叠，不可删除。

内置个人场景预设包括：

- 个人头像预设。
- 个人壁纸预设。
- 个人电商图预设。
- 个人产品海报预设。
- 公众号封面个人预设。
- 视频封面个人预设。
- 小红书封面个人预设。
- 内容头图个人预设。
- 短视频竖版个人预设。

保存为个人模板：

1. 调好提示词和参数。
2. 在模板区域填写模板名称和描述。
3. 点击“保存为个人模板”。

## 个人 Agent 优化

在“编辑提示词”中点击“帮我优化”可以优化当前提示词。

Agent 会返回：

- 优化后提示词。
- 优化后负面提示词。
- 与原提示词的对比说明。

点击“使用这版”后，会同时替换提示词和负面提示词，并自动收起候选区。

## Agent 对话

切换到“个人 Agent”后，可以像聊天一样让 Agent 协助你的个人生图流程。

Agent 可以提出需要确认的动作：

- 修改表单参数。
- 应用模板。
- 创建个人模板。
- 展示历史结果。
- 使用建议提示词并生成。
- 解释参数、结果或失败原因。

所有会改变表单、创建模板或触发生图的动作，都需要你点击确认后才会执行。

## Agent 图片上传

Agent 对话支持上传图片进行分析。

规则：

- 支持 PNG、JPEG、WEBP。
- 不支持 GIF 和 SVG。
- 单条消息最多 8 张图片。
- 本地保存不压缩、不转码。
- 发送给 Agent 时会转为 base64 data URL。
- 单张图片超过 10MB 时会弹出确认提示。

图片操作：

- 点击缩略图可打开大图预览。
- 已发送图片可以点击“加入参考图”，会切换到图生图模式并加入参考图列表。

## Agent 多会话

Agent 对话支持多个本地会话：

- 新对话：创建空会话。
- 我的会话：打开弹窗查看和切换会话。
- 重命名：修改会话标题。
- 删除：删除会话和该会话引用的图片资产。
- 清空当前对话：只清空当前会话消息和图片资产。

会话保存在 IndexedDB，刷新页面后仍会保留。

## 设置和清理

设置中可以：

- 保存或清除密钥。
- 切换 Agent 模型。
- 调整生成并发上限。
- 清空历史与历史图片。
- 清理未引用资源。
- 清空模板。
- 清除全部本地数据。

“清理未引用资源”只删除没有被我的历史或 Agent 对话引用的图片资产，不会破坏仍能回显的历史结果、参考图或对话图片。

建议长期使用时定期清理历史和未引用资源，避免 IndexedDB 占用过大。

## 常见问题

### 为什么新生成时旧结果不显示？

这是当前设计：结果区只显示当前批次。旧结果保存在我的历史里，点击历史即可显示。

### 为什么批量生成会提示确认？

超过 20 张图片会产生较多请求和费用，也更容易触发限流，所以会先确认。

### 为什么 Agent 上传大图会提示确认？

Agent 请求会把图片转成 base64 data URL，大图会让请求体变大，可能更慢或失败。

### 为什么刷新后运行中任务会继续？

刷新会中断浏览器内原请求。应用会读取 running 历史，把未完成任务重新入队续跑，这可能产生新的接口请求。

### 密钥保存在哪里？

保存在当前浏览器 IndexedDB，不会写入项目代码或构建产物。清除浏览器站点数据会删除密钥。

### 清理未引用资源会删除历史图片吗？

不会。它只删除没有被我的历史或 Agent 对话引用的孤立图片资产。要删除历史图片，需要删除对应历史或使用“清空我的历史”。
